# FAQ

## What is RelayWorks Document Processing Kit?

RelayWorks Document Processing Kit is a commercial source-code product for developers who need a self-hosted PDF-to-AI document processing workflow.

## What does the paid kit include?

The paid kit includes FastAPI and Next.js source code for converting PDFs into AI-ready outputs such as Markdown, HTML, JSON, plain text, processing reports, extracted images, and downloadable packages.

## Is the full source code included in this repository?

No. This public repository is only a showcase and product-evaluation hub. The paid source code is distributed separately after purchase.

## Where is the paid source code delivered?

The commercial source code is delivered through the [RelayWorks Gumroad product page](https://calebroge5.gumroad.com/l/djzlrg).

## Why is there no open-source license?

RelayWorks is a commercial product, not an open-source release. This showcase does not license the paid application source code.

## Does processing happen locally?

Yes. The product is designed for local and self-hosted processing so buyers can keep documents on their own machine or infrastructure.

## Does RelayWorks upload documents to a cloud service?

RelayWorks is designed for local and self-hosted processing, not as a hosted document-upload service. Buyers control the environment where it runs.

## What file types are currently supported?

The current product is PDF-first and does not claim support for every document format.

## Does it include Marker?

No. Marker is a third-party dependency and must be installed separately.

## What outputs are generated?

The workflow can generate Markdown, HTML, structured JSON, plain text, extracted images, a processing report, a selected goal output, and a complete ZIP package. Actual outputs depend on the source PDF.

## Can buyers modify the code?

Buyers receive the commercial FastAPI and Next.js source code to inspect and adapt, subject to the terms supplied with the paid product.

## Is this a SaaS?

No. The current kit is not a hosted SaaS product with accounts, billing, cloud storage, or multi-tenant operations.

## Does it include authentication?

No production authentication capability is claimed for the current kit.

## Does it support scanned PDFs?

Scanned PDFs may require OCR or additional handling. OCR accuracy is not guaranteed, and results depend on the document and local conversion tooling.

## Is extraction perfect?

No. Extraction quality depends on the PDF, layout complexity, embedded text, scans, images, tables, and underlying conversion tooling.

## What operating systems are recommended?

macOS or Linux is recommended for the local developer workflow.

## Is Docker included?

Docker is not claimed as part of the current product requirements. Buyers should plan for a local Python, Node.js, npm, Marker, and filesystem-based setup.

## What support is included?

Consult the current [Gumroad product page](https://calebroge5.gumroad.com/l/djzlrg) for the support terms offered with a purchase; this showcase does not promise additional support terms.
